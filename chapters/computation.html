4.2. Binary#

Computers are at their core dealing with ones and zeroes. This is known as ‚Äúbinary‚Äù. Binary code that the processor (CPU) understands is known as ‚Äúmachine code‚Äù. Machine code is usually thought of as the language at the lowest level. However, processors themselves sometimes use what‚Äôs known as ‚Äúmicrocode‚Äù but that‚Äôs too close to the hardware for this book. We‚Äôll talk more about high- and low-level programming languages when we get to Compilation [Chapter 8].

Binary counting existed before computers entered the scene. Let‚Äôs spend a few moments to get a better understanding of binary counting. Let‚Äôs consider two other number systems that you‚Äôre already probably familiar with: Tally marks and the Hindu-Arabic numeral system. Note that the Hindu-Arabic system is our normal decimal system without the decimals.

The system of tally marks is utterly trivial and I presume that most people recognize it from childhood. Zero is represented by no marks, and one is represented by a single mark. To add one to whatever number you already have, you simply add another mark. The number of marks you have determines the number that you have. When we tally something we tend to draw every fifth mark diagonally over the last four marks to make counting easier.

Let‚Äôs now talk about our good old Hindu-Arabic numeral system, which is also known as the base-ten positional numeral system. In the decimal system we have ten unique symbols representing the numbers 0 to 9. When we reach the last symbol, 9, and want to add one, we simply reset the number to 0 and add a 1 to the preceding digit. This process is simpler to grasp if you think of the number 9 as actually being 09.

\[\begin{split} 00 + 01 = 01\\ 01 + 01 = 02\\ 02 + 01 = 03\\ ...\\ 09 + 01 = 10\\ ...\\ 19 + 01 = 20\\ \end{split}\]

We say that this is a positional numeral system because the position of the digits matter. We say that this numeral system has base ten because each position is multiplied by a multiple of 10. See the algorithm?

\[ 123 = 1 * 10^2 + 2*10^1 + 3*10^0 \]

Digits in the first position from the right are multiplied by \(10^0\). Digits in the second position from the right are multiplied by \(10^1\). Digits in the third position from the right are multiplied by \(10^2\). And so forth.

Now let‚Äôs talk about binary. Binary is simply another positional system but with a base of two instead of ten. This means that the position of a digit determines what multiple of two that the digit should be multiplied with.

Let‚Äôs consider the number 12 in base ten. In binary, meaning in base two, we would express this as \(1100\). Why? Well‚Ä¶ The first two digits from the right are zeroes so we can disregard these. The third digit from the right however is a one, so it gives us:

\[ 1*2^2 = 4 \]

and the fourth digit from the right is also a one which gives us:

\[ 1*2^3 = 8 \]

which means that we end up with:

\[ 8 + 4 + 0 + 0 \]

which is equal to \(12\).

Since, in binary we either have or don‚Äôt have a position we can actually simplify this calculation significantly by pre-calculating what number each position either does or does not give us.

Rings a bell? :) You‚Äôve probably seen this number sequence somewhere before since it‚Äôs everywhere in computing. So, if we‚Äôve got a 1 in positions 2 and 3 (from right to left) then we‚Äôve expressed the number 3 since we end up with \(2 + 1\) or more explicitly \(0 + 0 + 0 + 0 + 0 + 2 + 1 + 0\).

In theory we can of course have any number of digits but in the example above we used 8. Why? Because each of these digits is known as a ‚Äúbit‚Äù and if you have eight bits this gives you a ‚Äúbyte‚Äù.

You might be familiar with the fact that we measure storage space in bytes. Let‚Äôs make more sense of this. When we say that the size of a file is 100 bytes we‚Äôre actually saying that to store the file we need 800 bits.

Just stop for a moment and contemplate how extremely impressive all this is. Let‚Äôs say we want to communicate information equivalent to one measly gigabyte. That‚Äôs not really an extreme size given that we‚Äôve got SD cards measured in terabytes. A gigabyte is equal to \(10^9\) bytes which in turn is equal to \(8 * 10^9\) bits. Meaning 8 billion on the short scale.

4.3. Transistors#

Modern processors contain billions of transistors. A transistor can be thought of as being turned on or off, which means that it can communicate a single bit, a zero or a one. To represent a byte we need eight transistors, since a byte is eight bits. So to communicate 100 bytes we need 800 transistors.

If you don‚Äôt consider cramming billions of transistors into one processor mind blowing I don‚Äôt know what could possibly be. How in the world can all this physically fit into a laptop case?
