When executed, the program will run the statement and print the string "Hello world" to screen. string is a data type, and we‚Äôll talk about Data types in a separate chapter but for now you can think of it this way: All values have types. Meaning that everything is a kind of thing. One kind of thing that values can be is string. A string is a sequence of characters.

Let‚Äôs look at another value. How about the numbers 42 and 3.14?

While we haven‚Äôt talked about Methods yet, WriteLine is a method. It is a method that is Overloaded so that it can accept values of multiple different types. This means that we can run the method with the string "Hello world" as well as with the Integer 42 and with the Double 3.14.

42
3.14

When we call the method WriteLine and pass the arguments "Hello world", 42, and 3.14 we are using what‚Äôs known as value ‚Äúliterals‚Äù. We are constructing these values out of thin air. If I speak the value 42 into existence twice you might ask yourself whether these two values are the same or not. If you write a program that asks whether 42 is equal to 42 and prints the result you will see that they indeed are the same, since we get the answer True.

True

By the way, the type of True is Boolean. You might recall this type from the chapter on Sets. The double-equal sign symbol (==) is an Operator and 42 == 42 is an Expression. We‚Äôll get to all these things soon but I‚Äôm just giving you some terminology now for context but I‚Äôm just giving you the correct terminology in passing.

The reason I wanted to highlight that 42 is equal to 42 is that this is not always the case. The reason that the values are the same in this case is that integers have value type semantics. It‚Äôs too soon to talk about that, but I just wanted you to beware that just because two values look like they are exactly the same doesn‚Äôt mean that they actually are.

Before we move on to the next chapter I want to show you that there‚Äôs one special value that‚Äôs the source of much pain and suffering in programming. This value denotes the absence of a value and is in C# known as null, but is in some other languages known as nil.

Prior to (C# 8.0), data types all reference types were, what is known as, ‚Äúnullable‚Äù. This meant that null was considered a valid value of all reference types. We will discuss this further in the chapter on Data types, null. Let‚Äôs, for example, say that you‚Äôve got a string, meaning a sequence of characters, but that your string is nullable. The compiler isn‚Äôt actually guaranteeing that you have a string, it is guaranteeing that you either have a string, or you have the absence of one (meaning null).

On the type level there is no way of ensuring that you only take a particular path through the program if you actually have a string. We‚Äôll talk more about null and nullable reference types in the chapter on data types.
